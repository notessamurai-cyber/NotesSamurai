# Production Checklist

> Master pre-generation checklist for NotesSamurai visual production.
> Run this checklist before any image generation begins for a song. References `Master.md`, `StyleGuide.md`, and the song's storyboard file (e.g. `finalsceneplan.md`).

---

## 1. Story Validation

- [ ] Storyboard follows the arc: Beginning → Conflict → Decision → Transformation → Hope (per `StyleGuide.md`)
- [ ] Every lyric section is mapped to a scene
- [ ] No scene contradicts the song's stated theme
- [ ] Ending resolves toward hope, not despair
- [ ] Scene count falls within the 14-16 target range unless justified

## 2. Visual Validation

- [ ] All scenes conform to Art Style rules in `Master.md`
- [ ] No forbidden elements present (fantasy, medieval, sci-fi armor — see `Master.md` Environment Rules)
- [ ] Compositions are minimal, no visual clutter
- [ ] Aspect ratio and target resolution confirmed (16:9, 3840x2160)

## 3. Character Consistency

- [ ] Protagonist matches locked appearance (hair, eyes, outfit, silhouette, accessory)
- [ ] No unapproved design variants introduced
- [ ] Any approved variant is logged in `Master.md` exceptions
- [ ] Character grows emotionally, not visually, across scenes

## 4. Emotional Progression

- [ ] Core Emotion sequence present: Loneliness → Reflection → Hope → Momentum → Transformation
- [ ] Each scene's `Story Purpose` is distinct — no redundant emotional beats
- [ ] Emotional intensity escalates logically toward the climax scene

## 5. Scene Pacing

- [ ] Each scene duration falls within 8-15 seconds
- [ ] No unnecessary scene changes (per storyboard efficiency requirement)
- [ ] Total runtime matches song length within acceptable tolerance
- [ ] Reused/merged assets (e.g. master chorus assets) are flagged and justified

## 6. Camera Variation

- [ ] No two consecutive scenes use the identical camera type
- [ ] Full camera vocabulary from `Master.md` is represented across the storyboard
- [ ] Camera choice matches the emotional intent of each scene

## 7. Color Progression

- [ ] Scenes follow the Color Script: Night → Blue Hour → Purple → Orange → Golden Sunrise
- [ ] No color stage skipped without narrative justification
- [ ] No mixed color stages within a single frame

## 8. Symbol Consistency

- [ ] Only approved recurring motifs used (confirm per-project motif list, e.g. Rain / Skyline / Sunrise)
- [ ] Each symbol's use matches its defined emotional meaning (`StyleGuide.md` Symbol Library)
- [ ] No new symbols introduced without documentation update

## 9. Prompt Validation

- [ ] Every scene has a corresponding entry in `PromptPack.md`
- [ ] Prompts describe illustration only — no camera movement language
- [ ] Standard Negative Prompt block applied to every prompt
- [ ] Prompt language matches `Master.md` Prompt Structure template

## 10. Image Generation Readiness

- [ ] Reference images identified where needed (character sheet, prior approved scenes)
- [ ] Model/tool for generation confirmed
- [ ] `GenerationLog.md` entry created for each scene prior to first attempt

## 11. Editing Readiness

- [ ] Each scene has a corresponding entry in `EditingNotes.md`
- [ ] Motion plan (zoom/pan/parallax) confirmed compatible with static-frame composition
- [ ] Transition type confirmed for every scene-to-scene handoff

## 12. Final Approval

- [ ] All scenes marked Approved in `GenerationLog.md`
- [ ] Creative Director sign-off recorded
- [ ] Storyboard file frozen (no further edits without version increment)
- [ ] Ready to proceed to `ReleaseChecklist.md`
